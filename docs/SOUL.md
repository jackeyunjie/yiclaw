## 一、核心使命

**让每个人都拥有一个永不疲倦、持续进化的 AI 员工**

OpenClaw 不是聊天机器人，不是问答工具。

OpenClaw 是**数字员工**——它像人类员工一样：
- 记住你的偏好和历史（长期记忆）
- 主动完成任务（定时任务 + 工具系统）
- 在不同场景下与你协作（多通道接入）
- 持续学习成长（Agent 进化）

---

## 二、第一性原理（不可违背）

### 1. 记忆即身份
> 没有记忆的 AI 只是工具，有记忆的 AI 才是伙伴。

- 用户的一切交互都必须被记住
- 记忆是上下文，不是负担
- 忘记用户 = 背叛信任

### 2. 通道即界面
> 用户在哪里，OpenClaw 就在哪里。

- Web、微信、Telegram、飞书——都是同一个员工
- 通道只是皮肤，核心能力一致
- 不强迫用户改变习惯

### 3. 工具即能力
> AI 的价值在于行动，而不在于说话。

- 每个工具都是一个技能
- 工具可以组合，形成工作流
- 工具越多，员工越能干

### 4. 模型即引擎
> 不绑定单一模型，只选择最适合的。

- 模型是耗材，不是核心
- 根据场景智能路由模型
- 永远可以替换、升级、降级

### 5. 安全即底线
> 用户的信任是 OpenClaw 存在的根基。

- API Key 加密存储
- 数据隔离严格
- 审计日志完整
- 不滥用用户数据

## 五、决策原则

当面临技术或产品决策时，问自己：

1. **这对记忆系统有何影响？**
   - 如果是削弱记忆，拒绝
   - 如果是增强记忆，考虑

2. **用户是否需要改变习惯？**
   - 如果是，能否提供迁移路径？
   - 最佳方案：用户无感知升级

3. **这是否让 AI 员工更有能力？**
   - 新工具 > 新界面
   - 自动化 > 交互式

4. **是否锁定特定供应商？**
   - 如果是，寻找抽象层
   - 保持替换自由

5. **用户数据是否安全？**
   - 默认加密
   - 最小权限
   - 可审计
