# OpenClaw 本地试用检查清单

## ✅ 启动前检查

- [ ] Node.js 18+ 已安装 (`node -v`)
- [ ] pnpm 已安装 (`pnpm -v`)
- [ ] 环境变量 `.env` 已配置
- [ ] 数据库已就绪（PostgreSQL 或 SQLite）

## 🚀 启动服务

```bash
# 在 openclaw 目录下执行
pnpm dev
```

等待看到以下日志表示启动成功：
```
🚀 Gateway server running on port 3000
🔗 Socket.io server ready
✅ 17 tools registered
```

## 🌐 访问系统

1. 打开浏览器访问：**http://localhost:5173**
2. 应该看到登录页面

## 📝 试用步骤

### Step 1: 注册账号
- [ ] 点击"注册账号"
- [ ] 填写用户名、密码
- [ ] 成功注册并自动登录

### Step 2: 配置 AI 模型
- [ ] 在顶部下拉框选择模型（如"通义千问"）
- [ ] 确保 `.env` 中配置了对应 API Key

### Step 3: 基础对话测试
```
输入: "你好，请介绍一下 OpenClaw"
检查: AI 是否正常回复
```

### Step 4: 工具调用测试
```
输入: "帮我查看当前目录有哪些文件"
检查: AI 是否调用 file_list 工具并显示结果
```

### Step 5: 长期记忆测试
```
输入: "记住我的目标是学习 TypeScript"
等待 AI 回复后
输入: "我之前设定的学习目标是什么？"
检查: AI 是否能回忆起之前的目标
```

### Step 6: 数据库工具测试
1. 点击顶部导航"数据库"
2. 执行 SQL: `SELECT * FROM "User"`
3. 检查：是否显示用户列表

### Step 7: 定时任务测试
1. 点击顶部导航"定时任务"
2. 创建任务：
   - 名称：测试任务
   - Cron：`* * * * *`（每分钟）
   - 指令：发送一条测试消息
3. 启用任务
4. 等待 1-2 分钟，查看执行日志

## 🎉 试用成功标准

完成以下即表示试用成功：
- [ ] 能正常注册登录
- [ ] AI 对话正常
- [ ] 至少一个工具调用成功
- [ ] 长期记忆功能正常
- [ ] 数据库查询正常

## ❌ 常见问题

### 问题 1: 端口被占用
```bash
# 结束占用 3000 端口的进程
netstat -ano | findstr :3000
taskkill /PID <PID> /F
```

### 问题 2: AI 无响应
- 检查 `.env` 中的 API Key
- 检查网络连接
- 查看后端日志

### 问题 3: 数据库错误
- 检查 `DATABASE_URL` 配置
- 运行 `pnpm db:migrate`

## 📞 需要帮助？

如果遇到问题：
1. 查看后端日志输出
2. 检查浏览器 Console 错误
3. 确认环境变量配置正确
